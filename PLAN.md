# 📈 오늘의 경제, 내 말로 — 제품 기획서 (PLAN)

> **한 줄 정의:** 환율·금리·물가·증시를 *경제 모르는 사람의 언어로* 매일 풀어주고,
> "그게 **내 돈**에 무슨 뜻인지"까지 알려주는 AI 경제 비서.

- **작업 코드명:** `econ-brief` (정식 브랜드명 미정 — 결정 필요 항목 참조)
- **문서 버전:** v0.3 (2026-06-22)
- **상태:** 핵심 결정 확정(§15) · 데이터 라이브 검증완료(§4) · **W1~W4 완료 + W5 운영안정화 완료** — 수집→생성→텔레그램 발송 **라이브 검증 완료**, 웹 아카이브·대출 알리미·스케줄러 동작. 남은 건 W5 지인 베타(실사용자)·COFIX 연동.
- **확정:** 첫 채널 = 이메일 뉴스레터 · 첫 타겟 = P1 영끌러 · 진입 훅 = 대출 금리 알리미(F2)
- **검증완료:** FRED · ECOS · KIS 시세 전부 ✅ — 남은 외부 의존성은 COFIX(은행연합회)뿐

---

## 1. 왜 이 제품인가

### 문제
- 경제 뉴스는 **기자의 언어**로 쓰여 있어 2030 일반인에겐 어렵고 멀다.
- "달러가 올랐다 / 기준금리 동결" → **그래서 내 삶에 뭐가 달라지는데?** 에 아무도 답을 안 해준다.
- 정보는 공개돼 있으나(한국은행·연준), 일반인은 **접근 → 해석 → 내 상황 적용**을 못 한다.

### 해결
- 공개 데이터(ECOS·FRED·KIS 시세)를 **매일 자동 수집** →
- LLM이 **친구가 설명하듯** 3줄 요약 + "너에게 무슨 의미" →
- 뉴스레터·웹·(이후)카톡으로 전달.

### 타겟 페르소나
| | 페르소나 | 핵심 니즈 |
|---|---|---|
| P1 | **불안한 영끌러** (29~40, 변동금리 대출 보유) | "내 이자 오르나?" — 금리 알리미 |
| P2 | **경제 입문 직장인** (25~35) | "뉴스 봐도 모르겠다" — 쉬운 일일 브리핑 |
| P3 | **해외직구·여행족** | "환전 지금 할까?" — 환율 타이밍 |

> P1을 **첫 타겟**으로(절박 = 리텐션·전환 좋음), P2로 **도달 확장**.

## 3. 핵심 기능

### F1. 매일 경제 브리핑 ⭐ (메인)
- 매일 아침 7:00, 전일 한·미 지표 + 증시를 **3블록**으로:
  1. **무슨 일이 있었나** (사실, 2~3줄)
  2. **왜 그랬나** (맥락, 1~2줄)
  3. **너에게 무슨 의미** (생활 연결: 대출·환전·물가, 1~2줄)
- 채널: 이메일 뉴스레터(MVP) → 웹 아카이브 → 카톡 채널 → 앱.

### F2. 내 대출 금리 알리미 🔔 (리텐션 핵심)
- 사용자 입력: 대출 유형(변동/혼합), 잔액, 금리 기준(COFIX/CD/국고채), 현재 금리.
- COFIX·시장금리 변동 감지 → **"네 월 상환액 ±N원 변동 예상"** + 쉬운 설명 푸시.
- 규칙 기반 계산(LLM 호출 최소화) + 설명 문장만 LLM.

### F3. 환율 타이밍 💱 (확장)
- 원/달러 추세 + 목표 환율 알림. "지금 살까/기다릴까"는 **정보 톤**(추천 아님).

> **MVP = F1 + F2.** F3는 v1.1.
>
> **P1 영끌러 우선 포지셔닝(확정):** 랜딩·온보딩은 **"대출 이자 오르면 알려드림(F2)"**을 전면에.
> F2 = 가입 이유(훅), F1 = 매일 여는 이유(습관). 금리 변동은 드물게 터지니, 그 사이를 일일 브리핑이 채운다.

## 5. 시스템 아키텍처

```
[스케줄러 06:30]
   │
   ├─ Collector ── ECOS / FRED / KIS  ──► [Postgres: raw 시계열]
   │                                           │
   ├─ Builder ──── 전일 대비·변동 계산 ◄────────┘
   │                  │
   ├─ Composer ──── LLM(Gemini)로 브리핑 생성 ── 검수(OpenAI 폴백)
   │                  │
   │             [Postgres: briefings]
   │                  │
   └─ Dispatcher ── 이메일 / 웹 / (카톡) 발송 ── 발송이력 적재
                      │
                 [Slack: 운영 알림]

[알리미 워커] COFIX/시장금리 변동 감지 → 영향 사용자 추출 → 푸시
[웹/API] FastAPI + Next.js (브리핑 아카이브 · 구독 · 알리미 설정)
```

## 7. LLM 브리핑 프롬프트 설계 (초안)

**시스템 역할**
- "너는 경제를 1도 모르는 친구에게 카톡으로 설명하는 친한 형/누나다."
- 규칙: 전문용어 금지(쓰면 즉시 풀이) / 단정적 투자 추천 금지 / 숫자엔 항상 '그래서 의미' 동반 / 과장·공포 조성 금지.

**입력(구조화 JSON)**
```json
{
  "date": "2026-06-20",
  "kr": {"기준금리": 3.0, "원달러": 1378, "원달러_전일대비": +6, "코스피": 2710, "코스피_등락": -0.4},
  "us": {"연방기금": 4.5, "미10년물": 4.2, "전일": "..."},
  "events": ["...주요 일정..."]
}
```

**출력 포맷(고정 3블록)** → 파싱·아카이브·길이 제어 용이.
> 비용·일관성 위해 **출력은 항상 같은 스키마**로 받고, 프론트에서 렌더.

## 8. 데이터 모델 (Postgres 초안)

```sql
-- 원천 시계열
indicators(id, source, series_code, name, unit)
indicator_values(indicator_id, ts, value, PRIMARY KEY(indicator_id, ts))

-- 브리핑
briefings(id, date UNIQUE, payload_json, body_md, model, created_at)

-- 사용자 / 구독
users(id, email, channel, verified_at, created_at)
subscriptions(user_id, type)            -- daily_brief / fx / loan

-- 대출 알리미
loan_profiles(id, user_id, kind, balance, benchmark, spread, current_rate, updated_at)
alerts(id, user_id, type, payload_json, sent_at)

-- 발송 이력
deliveries(id, user_id, briefing_id, channel, status, sent_at)
```

## 9. 개발 로드맵 (주차별)

| 주차 | 목표(완료 정의) |
|---|---|
| **W1 ✅** | ECOS·FRED 수집기 + Postgres 적재. "어제 vs 오늘" 변동 계산 동작. → **완료(2026-06-22):** 16개 지표 63,934행 적재, 변동계산·멱등성 검증. `docker compose` Postgres + `econ_brief/` 패키지. |
| **W2 ✅** | LLM 브리핑 생성 파이프라인. 매일 1건 자동 생성 → DB 저장 → 받기. → **완료:** `composer`(생성→저장) + `daily`(수집→생성→발송) + `telegram` **라이브 발송 검증 완료**(@gray_yoon_brief_bot) + `scheduler`(매일 07:00 KST). 슬랙/이메일 발송 모듈도 보유. |
| **W3 ✅(웹)** | 웹 아카이브 + 이메일 구독. → **웹 아카이브(`web.py` FastAPI) 완료:** 목록·상세·구독접수·JSON API 검증. 이중옵트인 확인메일만 추후 연결. |
| **W4 ✅** | F2 대출 알리미: 프로필 입력 → 벤치마크 변동 계산·설명. → **완료(`alerts.py`):** CD/국고채 경로 ECOS로 구현·검증(월이자 변화+LLM 설명+alerts 적재). COFIX는 은행연합회 연동 TODO. KIS 시세는 추후. |
| **W5 🟡** | 베타 오픈 + 피드백 + 운영 안정화. → **운영 안정화 완료(`util.py`):** 재시도(지수 백오프)·장애 텔레그램 알림 검증. 지인 베타는 실사용자 모집 필요. |
| W6+ | F3 환율, 카톡 채널, 개인화, 수익화 실험. |

> **W2 끝 = "매일 자동으로 내 메일함에 쓸만한 브리핑이 온다"** 가 첫 검증 마일스톤.
